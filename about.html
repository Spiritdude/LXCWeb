About LXCWeb

LinuX Containers (LXC) is a lightweight LINUX virtualization approach with little demands on the CPU unlike other approaches such as Xen.org or KVM; and therefore it is possible to run LXC containers within a virtual machine like KVM, VirtualBox or VMWare Player. The performance trade-off of this is lightweight virtualization is minimal as it's essentially just process and network isolation, using the same LINUX kernel.

LXCWeb is a simple web-interface to give control over LXC containers:

  • start, stop, freeze and unfreeze
  • create, clone and destroy
containers.

Additional there are two helpers included:

  • init-d-lxcweb which becomes /etc/init.d/lxcweb: it takes care if the host is (re-)booted that the containers which have been running before start again
  • lxcfarm: allows to create, clone, start, stop, freeze and unfreeze a farm of containers (when the web-interface is too cumbersome)

Naming Convention

I recommend following naming convention:
  1. create a base system, e.g. Ubuntu, create "ubu00" with "ubuntu" as type.
  2. start "ubu00", login (ssh) and update, install your favourite programs and databases.
  3. logout, and stop "ubu00"
  4. clone "ubu00" to "ubu01" by the web-interface, or call lxcfarm create -o ubu00 ubu01..30 creates 30 clones of ubu00
  5. start your "ubuXX", e.g. either via the web-interface or call on the command-line lxcfarm start ubu01..30 starts ubu01 to ubu30.

Updating LXCWeb

Visit LXCWeb @ Github and clone it from there, and call make install in the directory again.
